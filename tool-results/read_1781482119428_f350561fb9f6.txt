для&nbsp;читателей</a></li></ol></section></div>\n\n\n<h2 class=\"wp-block-heading\" id=\"id-0\">Что такое скиллы для&nbsp;ИИ-агентов&nbsp;— определение и&nbsp;зачем они нужны</h2>\n\n\n\n<p>Разработчики часто добавляют в&nbsp;проекты файл readme. Когда пользователь открывает этот файл в&nbsp;папке программы, он&nbsp;видит, что это за&nbsp;проект, как&nbsp;он&nbsp;устроен и&nbsp;как&nbsp;им&nbsp;пользоваться. Скиллы работают похожим образом. Skill&nbsp;— это инструкция для&nbsp;ИИ-агентов, по&nbsp;решению какой-то конкретной задачи. Она хранится в&nbsp;файле SKILL.md и&nbsp;находится внутри специальной папки.</p>\n\n\n\n<p class=\"relative\">По&nbsp;логике работы skills напоминают системные промпты и&nbsp;кастомные инструкции, но&nbsp;только на&nbsp;первый взгляд.<div class=\"lazyblock-link-aside-Z2aMsI5 wp-block-lazyblock-link-aside\"><a href=\"https://thecode.media/chto-umeyut-neyroseti-i-kak-vsyo-eto-ustroeno-otvechaem-na-chastye-voprosy-dlya-novichkov/\" target=\"_blank\" data-wpel-link=\"internal\">Что умеют нейросети и&nbsp;как&nbsp;всё это устроено: отвечаем на&nbsp;частые вопросы для&nbsp;новичков</a></div></p>\n\n\n\n\n\n<p><strong>Системные промпты.</strong> Это глобальные правила, которые нейросеть всегда применяет и&nbsp;не&nbsp;может игнорировать. Например, мы&nbsp;можем прописать правила: отвечай коротко, пиши понятно, ссылайся на&nbsp;источники в&nbsp;интернете. ИИ&nbsp;всегда будет эти правила соблюдать.</p>\n\n\n\n<p>Если&nbsp;вы&nbsp;настраивали проект (папку с&nbsp;чатами) внутри ChatGPT или&nbsp;других ИИ&nbsp;чат-ботов и&nbsp;прописывали для&nbsp;этого проекта инструкцию, то&nbsp;вы&nbsp;писали системный промпт.</p>\n\n\n\n<p><strong>Кастомные команды.</strong> Это сохранённые промпты, которые пользователь запускает через команду. Допустим, мы&nbsp;написали промпт, который анализирует длинный лог ошибки в&nbsp;коде и&nbsp;выдаёт короткое и&nbsp;понятное объяснение этой ошибки. Чтобы не&nbsp;писать или&nbsp;копировать этот промпт каждый раз мы&nbsp;настраиваем его вызов по&nbsp;команде: /error-summary. Вызов происходит только тогда, когда это действительно нужно, мы&nbsp;не&nbsp;применяем промпт к&nbsp;каждому ответу нейросети.</p>\n\n\n\n<p><strong>Скиллы.</strong> ИИ-агент, видит скиллы постоянно, как&nbsp;в&nbsp;случае с&nbsp;системными промптами, но&nbsp;применяет их&nbsp;не&nbsp;всегда, а&nbsp;только когда это уместно для&nbsp;задачи, как&nbsp;с&nbsp;кастомными командами. Нейросеть сама решает, когда применить скилл, если&nbsp;инструкция написана корректно, то&nbsp;вызывать его вручную не&nbsp;нужно. Навык можно считать качественным, если&nbsp;нейросеть сама вызывает его в&nbsp;90% случаев для&nbsp;выполнения релевантных задач.</p>\n\n\n\n<h2 class=\"wp-block-heading\" id=\"id-1\">Структура скилла&nbsp;— что внутри папки и&nbsp;файла SKILL.md</h2>\n\n\n\n<p>Скилл состоит не&nbsp;только из&nbsp;файла SKILL.md. Обычно он&nbsp;лежит внутри специальной папки. Там&nbsp;же находятся другие опциональные файлы. Это могут быть шаблоны, примеры хороших ответов, скрипты, которые ИИ&nbsp;должен сам запускать для&nbsp;решения задач.</p>\n\n\n\n<figure class=\"wp-block-table\"><div class=\"table-wrp\"><table class=\"has-fixed-layout\"><tbody><tr><td><strong>Файл/папка</strong></td><td><strong>Обязательно</strong></td><td><strong>Назначение</strong></td></tr><tr><td>SKILL.md</td><td>да</td><td>Здесь находится инструкция в Markdown + блок с объяснением для ИИ-агента, когда применять навык.</td></tr><tr><td>scripts/</td><td>нет</td><td>Исполняемый код (Python, Bash и др.) для автоматизации сложных задач.</td></tr><tr><td>references/</td><td>нет</td><td>Документация для ИИ, например, здесь могут быть примеры хороших результатов работы.</td></tr><tr><td>assets/</td><td>нет</td><td>Шаблоны, шрифты, иконки.</td></tr></tbody></table></div></figure>\n\n\n\n<p>У&nbsp;инструкции внутри SKILL.md тоже есть своя структура, без&nbsp;которой скилл будет работать некорректно.</p>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-2\">YAML frontmatter&nbsp;—&nbsp;метаданные для&nbsp;ИИ-агента</h3>\n\n\n\n<p>Вверху находится блок с&nbsp;метаданными. Здесь есть 2 важных поля name и&nbsp;description, они обязательные, без&nbsp;них скилл не&nbsp;будет нормально работать.</p>\n\n\n\n<p><strong>name </strong>—&nbsp;название навыка. По&nbsp;нему ИИ-агент отличает один скилл от&nbsp;другого.</p>\n\n\n\n<p><strong>description </strong>—&nbsp;описание задачи, для&nbsp;которой нужен скилл. Это подсказка для&nbsp;нейросети, с&nbsp;её&nbsp;помощью агент понимает, когда использовать навык.</p>\n\n\n\n<p>Чтобы ИИ&nbsp;увидел блок с&nbsp;name и&nbsp;description, он&nbsp;обязательно должен быть выделен тремя дефисами снизу и&nbsp;сверху. Такой формат записи называется <a href=\"https://thecode.media/json-v-python/\" target=\"_blank\" rel=\"noopener\" data-wpel-link=\"internal\">YAML frontmatter</a>.</p>\n\n\n\n<p>Пример блока с&nbsp;метаданными:</p>\n\n\n\n<div class=\"code-toolbar\"><pre class=\"wp-block-prismatic-blocks language-markdown line-numbers\" tabindex=\"0\"><code class=\"language-markdown\"><span class=\"token front-matter-block\"><span class=\"token punctuation\">---</span>\n<span class=\"token front-matter yaml language-yaml\">name: короткое название\ndescription: когда использовать скилл</span>\n<span class=\"token punctuation\">---</span></span>\nА здесь — инструкция в свободной форме: что делать, в каком порядке, чего избегать и как оформить результат.<span aria-hidden=\"true\" class=\"line-numbers-rows\"><span></span><span></span><span></span><span></span><span></span></span></code></pre><div class=\"toolbar\"><div class=\"toolbar-item\"><span>Markdown</span></div><div class=\"toolbar-item\"><button class=\"copy-to-clipboard-button\" type=\"button\" data-copy-state=\"copy\"><span>Copy</span></button></div></div></div>\n\n\n<div class=\"lazyblock-plashka-2i390k wp-block-lazyblock-plashka\">\n\n<p><strong>Полезный блок со&nbsp;скидкой</strong></p>\n\n\n\n<p>Скиллы помогают один раз описать рабочий процесс и&nbsp;больше не&nbsp;повторять агенту одни и&nbsp;те&nbsp;же инструкции. Anthropic выделяет 3 категории скиллов в&nbsp;своей <a href=\"https://docs.google.com/document/d/1lqSypCAD-PiyXT2CIq74jozCOMCtPZR629FO7HUQE44/edit?tab=t.0\" target=\"_blank\" rel=\"noreferrer noopener мы&nbsp;создаём самостоятельно для&nbsp;личного использования.</p>\n\n\n\n<p><strong>Навыки от&nbsp;организаций</strong>&nbsp;— когда у&nbsp;сотрудников компании есть корпоративные аккаунты и&nbsp;организация может предоставить им&nbsp;свои скиллы.</p>\n\n\n\n<p><strong>Навыки партнёров</strong>&nbsp;— скиллы в&nbsp;связке с&nbsp;<a href=\"https://thecode.media/ai-github-repositories-2026/\" target=\"_blank\" rel=\"noopener\" data-wpel-link=\"internal\">MCP</a> от&nbsp;крупных компаний, например, от&nbsp;Notion, Figma, Atlassian. Разработчики ИИ-агентов устраивают с&nbsp;ними коллаборации для&nbsp;повышения удобства.</p>\n\n\n\n<p>Ещё скиллы можно разделить на&nbsp;точечные и&nbsp;универсальные.</p>\n\n\n\n<p>Точечные скиллы хранятся в&nbsp;папках конкретных проектов, ИИ-агенты видят их&nbsp;только в&nbsp;рамках этих проектов. Например, у&nbsp;нас есть скилл для&nbsp;автоматизации агента по&nbsp;поиску вакансий в&nbsp;телеграм. Он&nbsp;запускает скрипт, собирает список новых вакансий за&nbsp;сутки и&nbsp;выводит отчёт с&nbsp;релевантными предложениями. Чтобы этот навык не&nbsp;захламлял память нейросети в&nbsp;других задачах, мы&nbsp;можем хранить его в&nbsp;папке проекта, ИИ-агент будет обращаться к&nbsp;нему, только при&nbsp;работе в&nbsp;этой папке.</p>\n\n\n\n<p>Универсальные скиллы хранятся в&nbsp;директории ИИ-агента, в&nbsp;специальной папке для&nbsp;скиллов. Нейросеть видит их&nbsp;всегда, независимо от&nbsp;проекта. Это удобно для&nbsp;решения универсальных задач. Например, есть скиллы для&nbsp;создания других скиллов. Этот навык может пригодиться в&nbsp;разных проектах, поэтому логично, чтобы у&nbsp;агента был всегда к&nbsp;нему доступ.</p>\n\n\n<div class=\"lazyblock-banner105-Z1sSbSH wp-block-lazyblock-banner105\">\n<a href=\"https://practicum.yandex.ru/catalog/programming/?utm_source=content&amp;utm_medium=brandmedia&amp;utm_campaign=kod_brandmedia_RF_Prog_Unde_b2c_Banner_None&amp;utm_content=15-06-2026&amp;utm_term=kak-sozdat-ai-skill-skill-md&amp;creative=banner3_progpaid\" class=\"third-type-banner\" target=\"_blank\" data-wpel-link=\"exclude\" rel=\"follow\">\n\t<img decoding=\"async\" class=\"third-type-banner__image\" src=\"https://thecode.media/wp-content/themes/thecode/assets/png/banners/third-type/banner-105.png\" width=\"558\" height=\"296\" alt=\"Полный гайд по&nbsp;AI-скиллам: что это такое и&nbsp;как создать скилл для ИИ-агента\" loading=\"lazy\">\n\t<div class=\"third-type-banner-content\">\n\t\t<span class=\"third-type-banner__subtitle typograf\">Обучение в&nbsp;Яндекс Практикуме</span>\n\t\t<span class=\"third-type-banner__text typograf\">Прокачайте скилл&nbsp;— смените грейд. Новые технологии, выше оффер</span>\n\t</div>\n</a></div>\n\n\n<h2 class=\"wp-block-heading\" id=\"id-5\">Как&nbsp;работает автотриггеринг скилла&nbsp;— почему агент сам выбирает навык</h2>\n\n\n\n<p>Скиллы не&nbsp;нужно вызывать вручную, это делает&nbsp;ИИ. Автотриггеринг навыков работает следующим образом:</p>\n\n\n<div class=\"small-list\">\n\n\n\n<ul class=\"wp-block-list\">\n<li>Мы&nbsp;даём ИИ-агенту задачу.</li>\n\n\n\n<li>Нейросеть проходится по&nbsp;скиллам из&nbsp;своего каталога и&nbsp;папки проекта, читает поле description.</li>\n\n\n\n<li>Сопоставляет данные из&nbsp;description с&nbsp;задачей.</li>\n\n\n\n<li>Если&nbsp;описание скилла относится к&nbsp;задаче, то&nbsp;ИИ-агент изучает дальнейшую инструкцию и&nbsp;выполняет&nbsp;её.</li>\n\n\n\n<li>Если&nbsp;описание в&nbsp;description не&nbsp;подходит для&nbsp;задачи, то&nbsp;нейросеть не&nbsp;читает инструкцию ниже и&nbsp;переходит к&nbsp;следующему скиллу.</li>\n</ul>\n\n\n</div>\n\n\n\n<p>Важно прорабатывать описание скиллов в&nbsp;description для&nbsp;триггера, оно должно быть конкретным. Допустим, у&nbsp;нас есть навык, который помогает учиться программировать и&nbsp;объясняет участки кода. Поле description можно описать так:</p>\n\n\n\n<p>description: Помогает с&nbsp;программированием.</p>\n\n\n\n<p>Это очень общая и&nbsp;размытая формулировка. ИИ-агент может решить, что это скилл для&nbsp;генерации скриптов и&nbsp;проигнорировать его, когда пользователь попросит объяснить&nbsp;код. Поэтому определение лучше прописать так:</p>\n\n\n\n<p>description: Используй этот скилл, когда пользователь учится программировать и&nbsp;просит простыми словами объяснить фрагмент кода, команду в&nbsp;терминале или&nbsp;сообщение об&nbsp;ошибке.</p>\n\n\n\n<p class=\"relative\">Это более конкретное описание. ИИ&nbsp;увидит его, поймёт, что этот скилл нужно использовать для&nbsp;объяснения кода, изучит инструкцию и&nbsp;корректно ответит пользователю.<div class=\"lazyblock-link-aside-Z1o61di wp-block-lazyblock-link-aside\"><a href=\"https://thecode.media/practicum/\" target=\"_blank\" data-wpel-link=\"internal\">Как&nbsp;устроен Практикум</a></div></p>\n\n\n\n\n\n<h2 class=\"wp-block-heading\" id=\"id-6\">Как&nbsp;создать скилл для&nbsp;агента&nbsp;— пошагово</h2>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-7\">Шаг 1&nbsp;— выбираем задачу</h3>\n\n\n\n<p>Первым делом надо выбрать задачу, которая регулярно повторяется. В&nbsp;целом установку навыков можно разделить на&nbsp;4 способа:</p>\n\n\n<div class=\"small-list\">\n\n\n\n<ul class=\"wp-block-list\">\n<li>Через команду агента.</li>\n\n\n\n<li>Через команду маркетплейса.</li>\n\n\n\n<li>По&nbsp;промпту.</li>\n\n\n\n<li>Ручное скачивание.</li>\n</ul>\n\n\n</div>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-17\">Установка через команду агента</h3>\n\n\n\n<p>Первый способ&nbsp;— установка через специальную команду ИИ-агента. Это может быть папка проекта или&nbsp;глобальная папка агента с&nbsp;другими навыками.</p>\n\n\n\n<figure class=\"wp-block-image size-full\"><img decoding=\"async\" width=\"940\" height=\"573\" src=\"https://thecode.media/wp-content/uploads/2026/06/unnamed-3-1.png\" alt=\"Полный гайд по&nbsp;AI-скиллам: что это такое и&nbsp;как создать скилл для ИИ-агента\" class=\"wp-image-66167\" loading=\"lazy\" srcset=\"https://thecode.media/wp-content/uploads/2026/06/unnamed-3-1.png 940w, https://thecode.media/wp-content/uploads/2026/06/unnamed-3-1-720x439.png 720w, https://thecode.media/wp-content/uploads/2026/06/unnamed-3-1-320x195.png 320w, https://thecode.media/wp-content/uploads/2026/06/unnamed-3-1-768x468.png 768w\" sizes=\"auto, (max-width: 940px) 100vw, 940px\"></figure>\n\n\n\n<h2 class=\"wp-block-heading\" id=\"id-21\">Частые вопросы об&nbsp;AI-скиллах</h2>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-22\">Чем скилл отличается от&nbsp;системного промпта?</h3>\n\n\n\n<p>Системный промпт задаёт общие правила работы агента: как&nbsp;отвечать, чего избегать, какой стиль использовать. Его правила применяются всегда.&nbsp; Скилл решает более узкую задачу, например, как&nbsp;делать код-ревью, писать документацию или&nbsp;собирать отчёт. Агент видит описание скилла и&nbsp;применяет инструкцию из&nbsp;него только тогда, когда это действительно нужно для&nbsp;решения задачи.</p>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-23\">Скиллы работают только с&nbsp;Claude или&nbsp;с&nbsp;другими агентами тоже?</h3>\n\n\n\n<p>Впервые skills появились в&nbsp;Claude Code, но&nbsp;сейчас их&nbsp;поддерживают Codex, Hermes и&nbsp;другие ИИ-агенты. У&nbsp;разных инструментов могут отличаться пути к&nbsp;папкам для&nbsp;хранения скиллов, но&nbsp;в&nbsp;остальном навыки идентичны и&nbsp;не&nbsp;привязаны к&nbsp;экосистеме какого-то одного агента. Например, мы&nbsp;можем добавить скилл от&nbsp;Claude в&nbsp;Codex, и&nbsp;он&nbsp;будет исправно работать и&nbsp;наоборот.</p>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-24\">Сколько скиллов можно установить одновременно?</h3>\n\n\n\n<p>Жёсткого универсального числа нет, всё зависит от&nbsp;конкретного агента и&nbsp;его настроек. Но&nbsp;устанавливать десятки скиллов на&nbsp;всякий случай&nbsp;— плохая идея. Агенту нужно видеть их&nbsp;названия и&nbsp;описания, чтобы выбрать подходящий. Чем больше лишних скиллов, тем больше они забивают память агента, это может запутывать нейросеть и&nbsp;приводить к&nbsp;ошибкам.</p>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-25\">Можно&nbsp;ли сделать скилл, который вызывает внешние&nbsp;API?</h3>\n\n\n\n<p>Да, если&nbsp;агент умеет запускать скрипты или&nbsp;пользоваться внешними инструментами. Главное&nbsp;— положить скрипты в&nbsp;директорию навыка, желательно в&nbsp;папку scripts. Ещё важно прописать инструкцию в&nbsp;SKILL.md и&nbsp;объяснить, как&nbsp;пользоваться скриптами и&nbsp;API.</p>\n\n\n\n<h3 class=\"wp-block-heading\" id=\"id-26\">Как&nbsp;проверить, что скилл правильно триггерится?</h3>\n\n\n\n<p>Проверьте его на&nbsp;нескольких похожих запросах: прямом, коротком и&nbsp;переформулированном. Если&nbsp;агент стабильно сам выбирает скилл, значит, описание написано понятно. Если&nbsp;скилл не&nbsp;срабатывает, доработайте description: сделайте описание более конкретным, чтобы агент понимал, когда использовать навык. Ещё можно прописать в&nbsp;SKILL.md правило, чтобы ИИ&nbsp;сообщал в&nbsp;ответе, о&nbsp;том, что использовал навык для&nbsp;выполнения задачи.</p>\n\n\n\n<p>Ai-скиллы для&nbsp;агентов&nbsp;— это способ один раз объяснить нейросети, как&nbsp;выполнять повторяющуюся задачу, и&nbsp;больше не&nbsp;расписывать одни и&nbsp;те&nbsp;же правила вручную. Главное&nbsp;— не&nbsp;пытаться запихнуть в&nbsp;него всё сразу: хороший скилл решает одну понятную задачу и&nbsp;точно описывает, когда его нужно применять. Чем яснее написан description, тем лучше агент поймёт, в&nbsp;какой момент подключить инструкцию.</p>\n\n\n\n<h2 class=\"wp-block-heading\" id=\"id-27\">Советуем дополнительно почитать
